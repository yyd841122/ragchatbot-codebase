## 方向 1：稳定性小修（优先级：高）

### 问题背景

基于 PR #19 的实际执行过程，发现以下不稳定因素：

1. **Stage 1 计划质量不稳定**
   - AI 生成的计划可能包含不准确或不存在的文件路径
   - 如果第一个文件不是真实存在的 README.md，会导致 Step 4 读取失败

2. **错误提示不清晰**
   - GitHub Actions 日志过长，关键信息不易找到
   - 用户难以快速定位失败原因

3. **执行失败无自动恢复**
   - Step 5 发现非 README.md 文件时会跳过，不会自动尝试其他文件
   - 执行失败时需要人工重新触发 `/zhipu-apply`

### 工作项

**1.1 改进 Stage 1 计划生成**

- **目标**：提高 AI 生成计划的文件路径准确率
- **方法**：
  - 优化 `agent_issue_handler.py` 中的 system prompt
  - 添加文件存在性检查逻辑（调用 GitHub API 验证）
  - 提供更明确的计划模板和示例
- **验证**：本地测试不同类型的 Issue

**1.2 添加简单的重试机制**

- **目标**：当第一个文件失败时，尝试从计划中提取下一个文件
- **方法**：
  - 在 `agent_issue_executor.py` 中添加重试逻辑
  - 当 Step 4 读取失败时，尝试下一个文件
  - 添加最大重试次数限制（3 次）
- **验证**：本地模拟失败场景

**1.3 改进错误提示**

- **目标**：让用户快速定位失败原因
- **方法**：
  - 在关键步骤添加清晰的日志输出
  - 在 GitHub Actions 日志顶部添加错误摘要
  - 在 Issue 回复中提供明确的失败原因和建议操作
- **验证**：本地模拟不同失败场景

### 预期效果（目标方向）
- Stage 1 计划准确率向 85%+ 方向优化
- 执行成功率向 80%+ 方向优化
- 用户排错时间向 3 分钟内方向优化

## 方向 2：文档入口整理（优先级：中）

### 问题背景

当前项目存在多个 Zhipu 相关文档，内容有重叠，新用户不知道从哪里开始：

- `ZHIPU_AGENT_USAGE.md`
- `ZHIPU_AGENT_QUICKSTART.md`
- `AUTOMATION_README.md`
- `PR_REVIEW_CHECKLIST.md`
- `STAGE6_MERGE_CHECKLIST.md`

### 工作项

**2.1 创建统一的用户入口文档**

- **目标**：为不同需求的用户提供清晰的文档导航
- **方法**：
  - 创建 `ZHIPU_GUIDE.md` 作为统一入口
  - 按用户场景组织内容（首次使用、日常使用、遇到问题）
  - 提供快速链接到具体操作文档
- **内容结构**：
  ```markdown
  ## 首次使用？
  → 快速开始：ZHIPU_AGENT_QUICKSTART.md

  ## 日常使用？
  → 使用指南：ZHIPU_AGENT_USAGE.md
  → Review 清单：PR_REVIEW_CHECKLIST.md

  ## 遇到问题？
  → 常见问题：ZHIPU_AGENT_USAGE.md#常见失败原因
  ```

**2.2 整合重复内容**

- **目标**：减少文档间的内容重复，提高可维护性
- **方法**：
  - 梳理现有文档，识别重复内容（如"当前限制"章节）
  - 将通用内容提取到独立文档
  - 在其他文档中引用而非重复

**2.3 归档过时文档（后续可评估）**

- **说明**：此为后续可评估的候选动作，本阶段先不执行大规模归档
- **方法**（如需执行）：
  - 创建 `docs/archive/` 目录
  - 将过时的测试脚本相关文档移入归档
  - 在归档文档顶部添加说明

### 预期效果
- 用户查找文档时间向 1 分钟内方向优化
- 新用户上手更容易
- 文档维护成本降低
