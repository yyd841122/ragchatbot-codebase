# 顺序工具调用实现计划（方案A - 循环架构）

## 上下文

当前RAG聊天机器人的工具调用功能只支持单轮执行，限制了AI处理复杂查询的能力。用户需要实现最多2轮连续工具调用的顺序工具调用功能，使AI能够：
- 处理多部分问题（如"搜索与课程X第4课讨论相同主题的课程"）
- 进行多个搜索比较
- 需要来自不同课程/课时信息时的复杂查询
- 基于前一个工具结果决定后续操作

**当前问题：**
- AI只能在一次API请求中使用工具
- 如果AI在看到工具结果后想要再次调用工具，无法做到
- 限制了处理复杂多步查询的能力

**预期结果：**
- AI可以最多进行2轮连续工具调用
- 每个工具调用作为独立的API请求
- AI能够推理先前的结果并决定是否需要更多工具
- 保持对话上下文，累积源信息，优雅处理错误

## 推荐方案：循环架构实现

采用循环架构重构 `_handle_tool_execution` 方法，使用显式 `while` 循环控制最多2轮工具调用。

### 架构优势
- 代码清晰，易于理解和调试
- 容易调整 MAX_ROUNDS 限制
- 终止逻辑集中在一处
- 符合传统编程模式

**新增方法1：`_execute_tools_round`**
```python
def _execute_tools_round(self, response, tool_manager) -> Tuple[Dict, List[Dict], List[Dict]]:
    """
    Execute all tool calls in a single round.

    Returns:
        Tuple of (assistant_message, tool_results_messages, sources_list)
    """
    assistant_message = {
        "role": "assistant",
        "content": response.choices[0].message.content or "",
        "tool_calls": []
    }

    tool_results = []
    sources = []

    for tool_call in response.choices[0].message.tool_calls:
        import json
        try:
            arguments = json.loads(tool_call.function.arguments)
        except:
            arguments = {}

        tool_result = tool_manager.execute_tool(
            tool_call.function.name,
            **arguments
        )

        assistant_message["tool_calls"].append({
            "id": tool_call.id,
            "type": tool_call.type,
            "function": {
                "name": tool_call.function.name,
                "arguments": tool_call.function.arguments
            }
        })

        tool_results.append({
            "role": "tool",
            "tool_call_id": tool_call.id,
            "content": tool_result
        })

        # 收集本轮执行的源
        round_sources = tool_manager.get_last_sources()
        if round_sources:
            sources.extend(round_sources)

    return assistant_message, tool_results, sources
```

**新增方法2：`_make_api_call`**
```python
def _make_api_call(self, messages: List[Dict], tools: Optional[List] = None) -> Dict:
    """
    Make API call with optional tools parameter.

    Args:
        messages: Complete message history
        tools: Tool definitions (include for intermediate rounds, exclude for final)

    Returns:
        API response object
    """
    api_params = {
        "model": self.model,
        "messages": messages,
        "temperature": 0,
        "max_tokens": 800,
    }

    if tools:
        api_params["tools"] = tools

    return self.client.chat.completions.create(**api_params)
```

**新增方法3：`_has_execution_errors`**
```python
def _has_execution_errors(self, tool_results: List[Dict]) -> bool:
    """
    Check if any tool executions failed.

    Args:
        tool_results: List of tool result messages

    Returns:
        True if any tool execution contained errors
    """
    for result in tool_results:
        content = result.get("content", "")
        error_indicators = ["error", "failed", "not found", "exception"]
        if any(indicator in content.lower() for indicator in error_indicators):
            return True
    return False
```

#### 1.2 重构 `_handle_tool_execution` 为循环架构

**文件：** `backend/ai_generator.py` (lines 100-167)

**新实现：**
```python
def _handle_tool_execution(self, initial_response, messages: List[Dict],
                          tool_manager, max_rounds: int = 2) -> str:
    """
    Handle sequential tool execution with up to 2 rounds.

    Args:
        initial_response: The response containing tool use requests
        messages: Existing message history
        tool_manager: Manager to execute tools
        max_rounds: Maximum number of tool calling rounds (default: 2)

    Returns:
        Final response text after tool execution
    """
    current_messages = messages.copy()
    current_response = initial_response
    current_round = 0
    tools = tool_manager.get_tool_definitions()

    # 循环处理最多max_rounds轮工具调用
    while current_round < max_rounds:
        # 终止条件1: AI不想要使用工具
        if not current_response.choices[0].message.tool_calls:
            break

        # 执行当前轮的工具
        try:
            assistant_message, tool_results, round_sources = self._execute_tools_round(
                current_response, tool_manager
            )

            # 更新消息历史
            current_messages.append(assistant_message)
            current_messages.extend(tool_results)

            # 终止条件2: 工具执行失败
            if self._has_execution_errors(tool_results):
                break

        except Exception as e:
            # 处理工具执行异常
            print(f"Tool execution error: {e}")
            break

        # 准备下一轮API调用
        current_round += 1
        if current_round >= max_rounds:
            break

        # 进行下一轮API调用（保持工具可用）
        current_response = self._make_api_call(current_messages, tools=tools)

    # 最终API调用（不包含工具）
    final_response = self._make_api_call(current_messages, tools=None)
    return final_response.choices[0].message.content
```

**新增方法 `accumulate_sources`：**
```python
def accumulate_sources(self):
    """Move last_sources to accumulated_sources"""
    for tool in self.tools.values():
        if hasattr(tool, 'last_sources') and tool.last_sources:
            # 避免重复
            for source in tool.last_sources:
                if source not in self.accumulated_sources:
                    self.accumulated_sources.append(source)
```

**修改 `get_last_sources` 方法：**
```python
def get_last_sources(self) -> list:
    """Get accumulated sources from all tool executions"""
    return self.accumulated_sources
```

**修改 `reset_sources` 方法：**
```python
def reset_sources(self):
    """Reset all accumulated and last sources"""
    self.accumulated_sources = []
    for tool in self.tools.values():
        if hasattr(tool, 'last_sources'):
            tool.last_sources = []
```

## 风险缓解

- **延迟增加**：每个额外轮次增加~1-2秒延迟 - 通过设置合理超时和监控缓解
- **令牌限制**：长对话可能达到令牌限制 - 通过监控令牌使用和实现消息截断缓解
- **AI无限循环**：AI不断请求工具 - 通过MAX_TOOL_ROUNDS硬限制和失败时终止缓解
- **向后兼容性**：更改可能破坏现有行为 - 通过功能标志和全面测试套件缓解
